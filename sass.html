Sass

O preprocesorze

Sass to preprocesor, który pozwala na włączenie do CSS takich rzeczy, jak zmienne, zagnieżdżanie, miksowanie, dziedziczenie itp. Jest to nadzbiór css, więc każdy kod css jest również prawidłowy. Obsługuje dwa rodzaje składni, scss i sass. SCSS jest nieco bardziej rygorystyczny, ponieważ zawiera nawiasy klamrowe i średnik, podczas gdy SASS jest silnie oparty na odpowiednich wcięciach.

Zmienne

Pomyśl o zmiennych jako o sposobie przechowywania informacji, które chcesz ponownie wykorzystać w swoim arkuszu stylów. Możesz przechowywać takie rzeczy, jak kolory, stosy czcionek lub dowolną wartość CSS, którą chcesz ponownie wykorzystać. Sass używa symbolu $, aby zmienić coś w zmienną. Oto przykład:

$font-stack:   Helvetica, sans-serif;
$primary-color : #333 ;
                    
body {
    font: 100% $font-stack;
    color: $primary-color;
}

Zagnieżdżanie

Pisząc HTML, prawdopodobnie zauważyłeś, że ma on wyraźną zagnieżdżoną i wizualną hierarchię. Z drugiej strony CSS nie. Sass pozwoli Ci zagnieździć selektory CSS w sposób zgodny z tą samą wizualną hierarchią kodu HTML. Pamiętaj, że nadmiernie zagnieżdżone reguły spowodują zawyżenie kwalifikacji CSS, które mogą okazać się trudne do utrzymania i są ogólnie uważane za złą praktykę.

nav {
    ul {
        margin: 0;
        padding: 0;
        list-style: none;
    }
  
    li { display: inline-block; }
}

Moduły

Nie musisz zapisywać całego Sassa w jednym pliku. Możesz go podzielić, jak chcesz, za pomocą reguły @use. Ta reguła ładuje inny plik Sass jako moduł, co oznacza, że możesz odwoływać się do jego zmiennych, mikserów i funkcji w pliku Sass za pomocą przestrzeni nazw opartej na nazwie pliku. Użycie pliku będzie również zawierało CSS, który generuje w skompilowanym wyniku!

// _base.scss
$font-stack: Helvetica, sans-serif;
$primary-color: #333;
 
body {
    font: 100% $font-stack;
    color: $primary-color;
}

// styles.scss
@use 'base';
                    
.inverse {
    background-color: base.$primary-color;
    color: white;
}

Mixiny

Niektóre rzeczy w CSS są trochę uciążliwe do pisania, szczególnie w przypadku CSS3 i wielu istniejących prefiksów dostawców. Mixin pozwala tworzyć grupy deklaracji CSS, które chcesz ponownie wykorzystać w całej witrynie. Możesz nawet przekazać wartości, aby Twój miks był bardziej elastyczny. Dobrym zastosowaniem miksera są prefiksy dostawców. Oto przykład transformacji.

@mixin transform($property) {
    -webkit-transform: $property;
    -ms-transform: $property;
    transform: $property;
}

.box { @include transform(rotate(30deg)); }

Rozszerzenia

To jedna z najbardziej przydatnych funkcji Sassa. Korzystanie z @extend umożliwia udostępnianie zestawu właściwości CSS z jednego selektora do drugiego. W naszym przykładzie zamierzamy stworzyć prostą serię komunikatów o błędach, ostrzeżeniach i sukcesach przy użyciu innej funkcji, która idzie w parze z rozszerzonymi klasami zastępczymi. Klasa zastępcza to specjalny typ klasy, która jest drukowana tylko wtedy, gdy jest rozszerzona i może pomóc w utrzymaniu skompilowanego kodu CSS w porządku i czystości.

%message-shared {
    border: 1px solid #ccc;
    padding: 10px;
    color: #333;
}
               
.message {
    @extend %message-shared;
}
